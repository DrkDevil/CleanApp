Apart from the rest.

Framesworks have been a part of developers, and designers arsenal of tools for a while now, and it's no surprise that they are used by millions of developers around the world to get help with jump starting their app. But there seems to be a consitant fundemntal flaw with todays Modern Frameworks. Massive Bloat! This is increased further after people start writting code ontop of these frameworks while creating thier apps, and websites.

Solving css class bloat!

To solve this issue we use what we call a variable design system. Unlike other frameworks BonsaiUI tries to limit the use of multiple classes in your HTML to style components and utilizes CSS Attributes heavily.

For example... Common frameworks use .alert to create the base alert, and the add .alert-yourcolor to style the alert. This seems a bit counter intuitive as it can bloat and crowd your code the larger your application gets over time. To solve the problem of using too many classes in your HTML to style one component BonsaiUI utilizes CSS Attributes heavily.

In BonsaiUI the .alert is the base class that creates your default unstyled alert. To add a color class simply add -primary, -info, or another color class listed in the variables styleguide directly to the .alert class. This will result in one class that looks like this .alert-primary

The disadvantage to this method is that it limits what names you can create for your own styles. For instance you would not be able to create a class call alertbox, because the BonsaiUI scss has declared that anyclass that starts with .alert will create the default alert box.

Why an App and UI?

BonsaiApp & BonsaiUI are two peas in the same pod. They work together to create a cohesive development platform for easy development. We use angular because of its reliability and it has been industy tested time and time again as a reliable platform for front-end development. Creating a seperate UI that works directly with our BaseApp is what helps create a fluid and consitant feel for the entire app.

Can I use a different UI with BonsaiApp?

Technically, Yes. You could easily remove all the scss files for BonsaiUI and include your own or another UI framework if you were so inclined. But the idea behind BonsaiUI is to give you the tools to easily create entire applications stylesheet within a single uniform enviormnet. This gives you more freedom as a designer to style your components exactly how you like.
