Introduction

Getting to know BonsaiApp & BonsaiUI.

What is BonsaiApp?

BonsaiApp is a Zen-like Angular Modular BaseApp with a Modern, & Elegant UI built with HTML 5, & SCSS for Rapid Application Development. BonsaiApp utilizes BonsaiUI Grid System, Angular Routing, and a highly organizied folder sturcture. All the things you need to quickly get started builing your Angular application.

Attribute Class Styling

Curing style bloat!

First off,what is this style bloat you speak of, and why is it such an issue?

Style bloat is a term BonsaiUI has kind of coined for the over use of classes to style HTML elements within a framework.

For example...

Several modern frameworks use a classes such as .alert to create a basic alert. But if you were to style it a bit further you might have to add another class such as .alert-color to style the alert. After that you may be adding more classes for positioning, or other utility classes that framework may be using. This seems a bit counter intuitive as it can bloat, and crowd your markup the larger your application gets over time.

In BonsaiUI the .alert is the base class that creates your default unstyled alert. To add a color class simply add -primary, -info, or another color class listed in the variables styleguide directly to the .alert class. This will result in one class that looks like this .alert-primary.

You can view the code behind the BonsaiUI Alerts here.

Unlike other frameworks BonsaiUI tries to limit the use of multiple classes in your HTML. So to solve the problem of "style bloat" in your HTML, BonsaiUI utilizes CSS Attributes heavily. We have also created an easy to use variable design system that controls the styles of every element in one highly organizied scss file.

Advantages of Attribute Class Stying

The Advantage to this method is that it allows you to utilize multiple classes as one. Thus limiting the need to clutter up your markup with loads of classes.

The Disadvantages

The disadvantage to this method is that it limits what names you can create for your own styles. For instance you would not be able to create a class called .alertbox, because the BonsaiUI has declared with attributes in the scss that any class that starts with .alert will create the default alert box.

Why an App & UI?

BonsaiApp & BonsaiUI are two peas in the same pod. They work together to create a cohesive development platform for easy development. We use angular because of its reliability and it has been industy tested time and time again as a reliable platform for front-end development. Creating a seperate UI that works directly with our BaseApp is what helps create a fluid and consitant feel for the entire app.

The future of BonsaiUI?

We plan on releasing a Vanilla version very soon, and already have an MVC .NET nuget in the works, with future plans for other Application Framesworks as well.

Can I use a different UI with BonsaiApp?

Technically, Yes. You could easily remove all the scss files for BonsaiUI and include your own or another UI framework if you were so inclined. But the idea behind BonsaiUI is to give you the tools to easily create entire applications stylesheet within a single uniform enviormnet. This gives you more freedom as a designer to style your components exactly how you like. And as a developer this give you the ease to create your application without worrying about style and consitency as it will already be implimented.
