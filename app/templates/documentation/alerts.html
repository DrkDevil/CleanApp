Alternative Colors & Links

Unlike other frameworks BonsaiUI tries to limit the use of multiple classes in your HTML to style components.


For example... Common frameworks use .alert to create the base alert, and the add .alert-yourcolor to style the alert. This seems a bit counter intuitive as it can bloat and crowd your code the larger your application gets over time. To solve the problem of using too many classes in your HTML to style one component BonsaiUI utilizes CSS Attributes heavily.


In BonsaiUI the .alert is the base class that creates your default unstyled alert. To add a color class simply add -primary, -info, or another color class listed in the variables styleguide directly to the .alert class.
This will result in one class that looks like this .alert-primary


The alert border is a transparent shade that matches the background color you choose automatically.

You can also create your own color styles easily in the alerts.scss.


The disadvantage to this method is that it limits what names you can create for your own styles. For instance you would not be able to create a class call alertbox, because the BonsaiUI scss has declared that anyclass that starts with .alert will create the default alert box.
